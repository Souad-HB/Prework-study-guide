HTML

  • HTML is made out of elements that work together to provide content. Elements such as text, image, header, list, etc.
  • HTML is the tool that builds the frame of your website
  • HTML is composed of two main elements; the head and the body
  • means this document was written is HTML
  • means the website language is written is English. the html is considered a "Tag" and the lang is considered and "Attribute"
  • "head" is considered an element and it includes the metadata
  • "meta" element that contains the information about the page
  • Difference between an element and a tag: Element refers to the block between the opening and closing tag, the Tag refers to what's between the angle brackets
  • "h1", "h2" are the headers
  • "p" is the paragraph
  • "ol" is an ordered list; usually numbered items
  • "ul" is an unordered list; usually bullet items
  • "li" is a list item within the ol or ul
  • "br" is a break; line of empty space
  • "a" is an anchor, includes link. followed by href attribute for the URL
  • "img" to add an image. src for the location of the image, alt is for the img description

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout main: switches to the main branch
  • git status: checks what branch we are currently on
  • pwd: to check where you currently are
  • cd folder-name: takes you to that folder
  • cd ..: takes you one location back
  • mkdir: creates a new directory in the location you are in. i.e: mkdir bootcamp
  • touch: creates a file in the location you are in. i.e: touch first-day.text
  • start: start first-day.text open the file
  • code .: open vs code of the branch you are in

Git Workflow

  1. make sure you are on the designated feature branch
  2. git add -A adds the changes files into the working directory and to eventually commit them to it
  3. git commit -m "the message describing your changes"
  4. git pull origin main: this is to double check if our local branch is in sync with the base branch in Github
  5. git push origin feature-branch-name: this pushes your changes to the remote branch
  6. Now you create the pull request from the Github browser
